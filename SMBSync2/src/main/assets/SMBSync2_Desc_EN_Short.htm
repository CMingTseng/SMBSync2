Function

To synchronize files between a PC/NAS and Android. It can be synchronized in one direction to the target from the master. Three Synchronization methods are available.（mirror,  move,  copies）

It is also possible synchronization of the SD Card and internal storage.*1

Can start the synchronization by external application (Tasker, AutoMagic etc) or SMBSync2 schedule.

 

        Mirror

If master and target files are different, to copied files from master to the target directory. To delete target files and directories if does not exist in the master.

        Move

If master and target files are different, to copied files from master to the target directory. To delete the master directories and files copied to the target.

However, the master and target same name, same file modification time and file size, to delete the file on the master without copying files.

        Copy

If master and target files are different, to copied files from master to the target directory. However, the same file modification time and file size with the same name does not copy the master and target.

 

Note1:

The difference file is one of the following three conditions.

1.      File does not exist

2.      Different file sizes

3.      Different over when last updated 3 seconds

 

Note2:

The last update time of the file sync to the internal storage may be rewritten to the sync time depending on the model. Also, the last update time of external SDCARD is always rewritten to synchronized time due to system limitation.

 

Note 3:

The SMB protocol supports only SMB1/SMB2. Since SMB 3 can not be used, please enable SMB1 or SMB2(2.01) with PC / NAS.

 

Note 4:

If you want to sync on a schedule sync or background in Android 6.0 or higher, please disable the optimization of the battery. (Menu -> Disable battery optimization)
